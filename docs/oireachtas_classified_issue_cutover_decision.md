# Oireachtas classified issue production cutover decision

**Status:** defer production cutover  
**Last updated:** 2026-06-30

## Decision

Do not repoint the production member-profile metrics workflow to the classified issue compatibility output yet.


## Reason to defer

The current classified issue compat file was generated by the enrichment trial workflow with the default trial row limit:

```text
row_limit: 50
source_rows: 47275
```

That is enough to validate shape and workflow wiring, but not enough for production speech metrics. Repointing production now would risk reducing issue/speech coverage.

## Required before production cutover

1. Run the classified issue enrichment trial with full source coverage:

```text
row_limit=0
```

2. Confirm DQ pass on full output.
3. Rerun member-profile metrics trial with the full compat file.
4. Compare speech metric coverage against production metrics:

```text
top_issue_2025
top_issue_count_2025
speech_count_2025
speech_rank_2025
```

5. Only then patch `.github/workflows/build_member_profile_metrics_2025.yml` to add:

```yaml
      DEBATE_ISSUES_INPUT_KEY: "processed/oireachtas_unified/compat/debates/debate_speeches_classified_compat.csv"
```

## Rollback if later cut over

Remove the `DEBATE_ISSUES_INPUT_KEY` override from the production workflow, or set it back to:

```text
processed/debates/debate_speeches_classified.csv
```

## Current conclusion

The compat route is structurally valid, but production cutover is deferred until a full-row classified issue compat build and speech-metric comparison are complete.
